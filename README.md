# 监装报销统计 · 单文件 HTML 台账

一个面向船舶监装业务的**费用报销台账**。整个应用是**一个 HTML 文件**,双击即用,不用安装、不用联网、不用后端——数据存在浏览器本地,一份流水自动生成人员 / 船东 / 利润多视图汇总。

> 起点是一张 44 个 Sheet、靠 VBA 宏同步 41 张人员分表的 Excel 台账。本项目把它重构成单文件 HTML,给一位非技术业务人员单机离线记账用:分发宏与分表消失,汇总实时自动算。

## 快速开始

下载 [`jianzhuang-ledger.html`](jianzhuang-ledger.html),双击用 Edge 或 Chrome 打开即可。文件内置一份**脱敏演示数据**(均为虚构船名 / 人名 / 金额),可随意增删试用。

## 功能特性

- **五页签**:台账流水(日常记账主页)/ 船次管理 / 按人员汇总 / 按船东汇总 / 设置与数据
- **一份流水,多视图自动汇总**:人员报销 / 补贴 / 天数、船东监装费 / 利润,实时计算
- **表单录入**:可搜索下拉选船次 / 人员,支持内联新建;科目、汇率、填表日期一应俱全
- **Excel 导出**:一键导出三张工作表;报销统计表按船号中的年份 / 月份 / 次数自然升序,按人员汇总按三位人员序号升序;合计行、列宽、冻结首行和数字格式已设好
- **JSON 备份 / 导入**:数据不随文件走,跨设备靠导出 / 导入 JSON 迁移;超过 7 天未备份主动提醒
- **版本更新检测**(v1.6 起):启动时向本仓库查询公开版本号(仅下载一个版本号文件,不上传任何数据),发现新版顶部横幅提示,一键"先备份再下载新版";无网络时自动失效,不影响使用
- **业务细节**:人员三位序号(001 起且不可重复)、外聘标记、归属人(可按归属人合并汇总)、可扩展报销科目、汇率清单快照、美元垫付独立标记
- **宽表体验**:台账主表横向滚动时「日期 + 船名」两列固定在左侧,不会认错行;每条船次起始行有分组强调条
- **低疲劳配色**:暖糖果主题,全面浅底深字,对比度达 WCAG AA

## 技术栈

- 单个 `.html` 文件,内嵌 [Vue 3](https://vuejs.org/) + [SheetJS](https://sheetjs.com/)(社区版),全部离线,无 CDN 依赖
- `localStorage` 本地持久化,`file://` 双击即用,零安装
- 数据模型:流水 `flow` + 船次 `voyage` 双表,人员 / 船东 / 科目 / 汇率为名录

## 隐私说明

**这是纯本地单机应用**——你录入的数据只保存在**当前电脑的浏览器里**,不会写回 HTML 文件本身,更不会上传到任何服务器。换电脑 / 备份 / 分享时,通过「设置与数据 → 导出 JSON 备份」迁移。

本仓库内的演示文件仅含**虚构的示例数据**,不含任何真实业务信息。

## 版本升级

应用启动时会检查本仓库的 `version.json`,发现新版本会在页面顶部提示。升级三步:

1. 点横幅上的「先备份再下载新版」(自动导出 JSON 备份并开始下载新文件,建议保存到旧文件同一文件夹);
2. **确认下载文件夹中已有刚导出的 JSON 备份文件**,再关闭旧页面——未看到备份文件,先用「导出 JSON 备份」重试,不要继续升级;
3. 打开下载的新文件确认:数据在则完成;若显示演示数据,请勿录入任何内容,在欢迎窗点「导入 JSON 备份」恢复刚导出的备份。

数据不丢失的保障主体是这份**经你确认存在的备份文件**。同一台电脑、同一 Windows 用户、同一浏览器且非隐私窗口下,新文件通常会自动沿用原数据,但此行为不作承诺;换电脑、换浏览器、清理浏览器数据后一律用「导入 JSON」恢复。

## 维护与发布规则(维护者)

**发布 checklist(顺序刚性)**:

1. 改 `JZBX_CORE.APP_VERSION`;
2. 脱敏终检:对最终 HTML 全文检索真实标识样本清单(清单仅存维护者本地,不入库);
3. commit + push HTML 到 main;
4. 建**正式** Release(tag `vX.Y`),上传 HTML 附件,文件名固定 `jianzhuang-ledger.html`;
5. 浏览器实测 `releases/latest/download/jianzhuang-ledger.html` 直链:核对下载附件顶栏版本、待发布 manifest 版本、Release tag 三者一致;
6. **最后**才 push `version.json`——保证用户看到升级提示时,制品一定已可下载且是新版。

**长期发布不变量**(对以后每次发布生效):

1. 每个可能成为 latest 的正式 Release 必须包含唯一的 `jianzhuang-ledger.html` 附件;
2. 预发布(prerelease)和草稿(draft)不得用于对外 `version.json` 所指版本;
3. push `version.json` 前必须完成第 5 步三方核对。
